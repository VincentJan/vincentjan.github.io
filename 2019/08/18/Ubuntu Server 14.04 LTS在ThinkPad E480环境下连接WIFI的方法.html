Ubuntu Server 14.04 LTS在ThinkPad E480环境下连接WIFI的方法

发表于 2019-08-18 | 分类于 笔记 ， 计算机 | | 阅读次数:
字数统计: 1.2k | 阅读时长 ≈ 5

前言：

一直以来想要建立一个linux系统下的工作环境，然而他的桌面版问题太多运行太慢难以满足需求。近来突发奇想想要直接安装一个Server版Ubuntu并且不安装桌面环境，用以进行简单的文本编辑和C语言学习，顺便熟悉一下linux的操作。

然而天公不作美，E480采用的rtl8821ce网卡驱动并不被这个版本的系统支持，故而开始了我的受苦之旅。费了三天时间，终于爬出坑来，于是写下这篇文章记录这次的经历，以备以后使用。

  • step1 安装操作系统

这一步已经轻车熟路没什么好说的了，只要注意语言选英文，并且最后是否自动更新那一步选不自动更新就好。

  • step2 更换软件源

ubuntu自带的软件源由于在国外，速度相当差强人意。为了提高速度，有必要对软件源进行更换。出于一种谜之心理，我选择了清华大学的源。操作如下。

cd /etc/apt
sudo mv sources.list sources.list.bak
sudo vim sources.list

将里面的内容改为

deb https://mirrors.tuna.tsinghua.edu.cn/ubuntu/ trusty main restricted universe multiverse
deb https://mirrors.tuna.tsinghua.edu.cn/ubuntu/ trusty-updates main restricted universe multiverse
deb https://mirrors.tuna.tsinghua.edu.cn/ubuntu/ trusty-backports main restricted universe multiverse
deb https://mirrors.tuna.tsinghua.edu.cn/ubuntu/ trusty-security main restricted universe multiverse

可在清华大学镜像站使用帮助|ubuntu获取完整镜像。

完成后进行如下操作

sudo apt-get update
sudo apt-get upgrade

  • step3 升级内核版本

Ubuntu Server 14.04 LTS 自带的内核版本是4.4，而安装rtl8821ce网卡驱动需要高于4.14的内核版本，需要注意内核版本并非越高越好，比如5.0就无法进行安装，并且内核降级比升级麻烦许多，故而升级需谨慎。

我选择升级到4.14版本，且需要4.12的过渡。假设文件在u盘上，操作如下：

sudo mount -t vfat /dev/sdb1 /media/usb
cd /media/usb/4.12
sudo dpkg -i *.deb
sudo reboot

此时一定要重新启动，并且使用

umane -sr

检查内核版本号是否已经成功升级。此时需要重新挂载u盘，然而并不能直接升级到4.14。不知道出于什么原因，系统会缺失libssl-1.1和linux-base_4.5两个package。导致安装无法进行，因此我们需要先从网上找到这两个包安装上再进行升级操作。

sudo mount -vfat /dev/sdb1 /media/usb
cd /media/usb/libssl1.1
sudo dpkg -i .deb
cd /media/usb/linuxbase4.5
sudo dpkg -i .deb
cd /media/usb/4.14
sudo dpkg -i *.deb
sudo reboot

以上安装顺序不能乱，这三次安装之间存在依赖关系。

现在系统内核就升级好了，进行下一步。

  • step4 安装GCC并且更改默认版本

首先安装gcc与g++

sudo apt-get install gcc g++

若出现错误还需安装依赖包（下次安装同理）

sudo apt-get install software-properties-common

默认安装的是4.8版本，而我们需要4.9以上版本，这里选择安装5.5，操作如下：

安装程序：

sudo add-apt-repository ppa:ubuntu-toolchain-r/test
sudo apt-get update
sudo apt-get install gcc-5 g++-5

更新链接：

sudo ln -s /usr/bin/gcc-5 /usr/bin/gcc -f
sudo ln -s /usr/bin/g++-5 /usr/bin/g++ -f

此时gcc默认为5.5版本，运行

gcc -v

确认版本。

  • step5 安装网卡驱动

经过了漫长的准备终于到了正题，要安装网卡驱动，首先将rtl8821ce.zip解压到任意位置，并且进入该目录：

cd /home/username/rtl
unzip rtl8821ce.zip

修改Makefile：

sudo vim Makefile

将其中的

export TopDIR ?= …

等号后内容改成Makefile的目录。

接着进行如下操作：

make
sudo make install
sudo modprobe -a 8821ce

此时驱动就安装好了，可以开始连接wifi了。

  • step6 Ubuntu Server下连接WiFi

网上试了很多方法大多以失败告终，最终只得退而求其次连接一个未加密wifi。

首先关闭禁止使用WiFi的限制

sudo apt-get install rfkill
sudo rfkill unblock all

然后打开/etc/network/interfaces, 在文件末尾加入

auto wlan0
iface wlan0 inet dhcp

保存并关闭文件，然后运行以下两条命令以重启网络服务

sudo ifdown wlan0
sudo ifup -v wlan0

此时可以连接WiFi了，运行

sudo apt-get install iw
sudo iw dev wlan0 scan |less

搜索可用WiFi列表，查找要连接的WiFi，然后运行

sudo iw dev wlan0 connect SSID

连接目标WiFi，并运行以下命令获取IP地址

sudo apt-get install dhcpcd
sudo dhcpcd wlan0

大功告成！

结束语

折腾了三天还是无法连接加密过的WiFi多少有点遗憾，然而这都是自己知识不够技术水平不到的结果，全程根着网上的博客照葫芦画瓢。这已经超出了我的能力范围，即便最后连上了也学不到什么东西，因此再纠结下去恐怕得不偿失。所以这次的受苦之旅就先结束吧。
